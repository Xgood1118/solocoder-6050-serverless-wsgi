在已有的「API Gateway 与 ALB 到 WSGI 请求转换」基础上新增「响应转换与可观测性」能力：现有 handle_request 已经能把入参事件转成标准 WSGI 请求交给 Flask 应用跑，需要补齐出站方向的语义；例如把 WSGI 响应体里大于阈值的二进制流自动落盘到 S3 并在响应头里返一个签名 URL；；在已有的「requirements 自动打包」基础上新增「lockfile 校验与多源依赖」能力：packRequirements 默认读 requirements.txt 单文件列表，需要支持读取 pip-compile 生成的 requirements.lock 作为权威源；CI 场景下若 lock 与 requirements.txt 漂移要报错阻断部署；同时允许通过 custom.wsgi.extraIndexUrls 配置私有 PyPI 源（多个），pip install 时按顺序注入且对 URL 做 host 白名单校验避免 SSRF；；在已有的「wsgi serve 本地启动」基础上新增「请求录制与回放」能力：本地 serve 启动时若传入 record 与目录路径参数则把每个请求的事件、上下文、响应全部按 messagepack 序列化写到该目录；replay 与目录路径参数启用时按时间序重放给本地 WSGI 应用；重放模式下需跳过真实网络监听并允许按录制时的 query string 哈希命中缓存跳过应用直接返录制时的响应体；；在已有的「wsgi exec 与 wsgi command 与 wsgi flask 与 wsgi manage 远程子命令」基础上新增「超时与资源配额」能力：远程调用默认无超时且不限制输出体大小，需要按子命令分别配置默认 timeoutSeconds（exec 30s、command 120s、flask 与 manage 60s）以及最大输出字节数（默认 5MB 超出截断并在末尾追加省略标记）；调用方可以通过 timeout 与 max-output 覆盖；且当远程 Lambda 因超时被杀时要把剩余 partial 输出透传给用户而非吞掉。
